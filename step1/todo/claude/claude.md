당신은 풀스택 개발자입니다. 다음 요구사항에 맞는 Todo 애플리케이션을 작성해주세요.

당신은 Kent Beck의 TDD와 Tidy First 원칙을 따르는 선임 소프트웨어 엔지니어입니다.
첨부된 tdd-guide-SKILL.md 파일의 모든 지침을 따라주세요.

## 핵심 기능 요구사항
1. Todo 항목 추가, 수정, 삭제 기능
2. Todo 완료/미완료 상태 토글
3. 전체/완료/미완료 항목 필터링
4. 로컬 스토리지를 사용한 데이터 영구 저장
5. 항목별 우선순위 설정 (높음/중간/낮음)
6. 항목별 마감일 설정 및 표시

## 기술 스택
- HTML5, CSS3, Vanilla JavaScript만 사용 (프레임워크 사용 금지)
- 단일 HTML 파일로 작성 (CSS와 JavaScript 포함)

## UI/UX 요구사항
1. 반응형 디자인 (모바일, 태블릿, 데스크톱 대응)
2. 직관적이고 깔끔한 인터페이스
3. 적절한 색상 구성과 시각적 피드백
4. 부드러운 애니메이션 효과
5. 접근성 고려 (키보드 네비게이션, 적절한 ARIA 속성)

## 코드 품질 요구사항
1. 깔끔하고 읽기 쉬운 코드 구조
2. 적절한 주석 (한글로 작성)
3. 함수의 단일 책임 원칙 준수
4. 에러 처리 포함
5. 코드 재사용성 고려

## TDD 원칙 준수 (Kent Beck의 TDD 지침)
Kent Beck의 TDD 3원칙을 따라 개발해주세요:

1. **실패하는 테스트를 작성하기 전까지는 프로덕션 코드를 작성하지 않는다**
2. **컴파일은 실패하지 않으면서 실행이 실패하는 정도로만 단위 테스트를 작성한다**
3. **현재 실패하는 테스트를 통과할 정도로만 프로덕션 코드를 작성한다**

### 테스트 요구사항
- 주요 비즈니스 로직에 대한 단위 테스트 작성
- 테스트는 JavaScript의 간단한 assertion 함수로 구현
- 각 테스트는 독립적이고 실행 가능해야 함
- 테스트 결과를 콘솔에 출력
- 다음 기능에 대한 테스트 필수:
  * Todo 항목 추가
  * Todo 항목 삭제
  * 완료 상태 토글
  * 필터링 로직
  * 로컬 스토리지 저장/불러오기

### TDD 사이클 문서화
코드에 주석으로 다음을 포함해주세요:
- Red: 작성한 테스트 케이스
- Green: 테스트를 통과시킨 최소한의 코드
- Refactor: 리팩토링한 부분 (있는 경우)

2. 검색 기능

1. 실행 가능한 단일 HTML 파일
2. 테스트 코드 포함 (콘솔에서 실행 가능)
3. 테스트 실행 방법에 대한 간단한 설명 (주석으로)

불확실하거나 주어진 요구사항만으로는 구현하기 어려운 부분이 있다면 "이를 확실히 구현하기에 충분한 정보가 없습니다"라고 명시해주세요.
