  TDD 원칙을 철저히 따라 Todo 애플리케이션을 작성했습니다. 주요 특징은 다음과 같습니다:

  📋 구현된 기능

  핵심 기능

  - ✅ Todo 항목 추가/수정/삭제
  - ✅ 완료/미완료 상태 토글
  - ✅ 전체/완료/미완료/우선순위별 필터링
  - ✅ localStorage를 통한 데이터 영구 저장
  - ✅ 우선순위 설정 (높음/중간/낮음)
  - ✅ 마감일 설정 및 표시 (지연 여부 표시)
  - ✅ 통계 정보 표시

  UI/UX

  - ✅ 반응형 디자인 (모바일, 태블릿, 데스크톱)
  - ✅ 깔끔하고 직관적한 인터페이스
  - ✅ 부드러운 애니메이션 효과
  - ✅ 접근성 (ARIA 속성, 키보드 네비게이션)

  🧪 TDD 개발 과정

  Kent Beck의 TDD 원칙을 따라 개발했습니다:

  Red-Green-Refactor 사이클

  1. RED: 실패하는 테스트 작성
  2. GREEN: 최소한의 코드로 테스트 통과
  3. REFACTOR: 코드 개선 (필요시)

  테스트 커버리지 (총 22개 테스트)

  - Todo 생성 테스트 (7개): 텍스트, 우선순위, 마감일, 기본값, ID, 검증
  - Todo 조작 테스트 (5개): 추가, 토글, 삭제, 예외 처리
  - 필터링 테스트 (5개): 전체, 완료, 미완료, 우선순위별
  - localStorage 테스트 (4개): 저장, 불러오기, 빈 데이터, 손상된 데이터
  - 통계 테스트 (1개): 전체/완료/미완료 개수 계산

  🚀 실행 방법

  1. todo-app.html 파일을 브라우저로 열기
  2. 페이지 로드 시 자동으로 테스트가 실행됨
  3. 하단의 "테스트 결과" 섹션에서 결과 확인
  4. 콘솔(F12)에서도 상세한 테스트 결과 확인 가능

  📝 코드 구조

  ├── 테스트 프레임워크 (assert, assertEqual, runTests)
  ├── 비즈니스 로직
  │   ├── createTodo() - Todo 생성
  │   ├── addTodo() - Todo 추가
  │   ├── toggleTodoCompletion() - 완료 토글
  │   ├── deleteTodo() - Todo 삭제
  │   ├── filterTodos() - 필터링
  │   ├── saveTodosToStorage() - 저장
  │   └── loadTodosFromStorage() - 불러오기
  ├── 단위 테스트 (5개 테스트 스위트)
  └── UI 렌더링 및 이벤트 핸들러

  🎯 TDD 원칙 준수

  - ✅ 모든 기능은 테스트를 먼저 작성
  - ✅ 테스트를 통과하는 최소한의 코드만 구현
  - ✅ 작은 단계로 점진적 개발
  - ✅ 각 테스트는 독립적으로 실행 가능
  - ✅ 명확한 테스트 이름으로 의도 표현
  - ✅ 코드 주석으로 TDD 사이클 문서화
